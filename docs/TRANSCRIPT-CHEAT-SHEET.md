# Transcript Recommendation Cheat Sheet

Use this as the fast crosswalk from transcript recommendations to the curriculum.

| Transcript recommendation | Where it appears |
| --- | --- |
| Start with setup, ergonomics, and repo warm-up before asking for serious work | `labs/00-setup-onboarding/LAB.md` |
| Teach terminal, IDE, and shell habits explicitly | `labs/00-setup-onboarding/LAB.md`, `docs/WORKFLOW-INTEGRATION.md` |
| Choose the right primitive instead of overusing a single prompt | `labs/01-primitives/LAB.md` |
| Plan before editing | `labs/02-planning/LAB.md` |
| Use Git history and issue archaeology to recover intent | `labs/03-history-issues/LAB.md` |
| Give Claude better evidence, not vaguer feedback | `labs/04-feedback-tools/LAB.md` |
| Use screenshots, tests, simulator traces, and CLI output as feedback artifacts | `labs/04-feedback-tools/LAB.md` |
| Add explicit review and verification gates | `labs/05-review-testing/LAB.md`, `templates/verification-gate-template.md` |
