# Detailed Syllabus

## Day 1 — The DNA of Claude Code

Participants learn the building blocks underneath Claude Code: context windows, transcripts, compaction, tools, skills, plugins, slash commands, hooks, MCP, agents, subagents, custom agents, background agents, workflows, goals, debate mode, and orchestration.

**Lab:** primitive design lab. Given a software task, decide whether it deserves a prompt, tool, skill, slash command, custom agent, subagent, hook, MCP server, background agent, or workflow.

## Day 2 — From Work to Implementation Plan

Participants take a piece of work and turn it into an execution-ready plan. Topics include intake assessment, bounded codebase Q&A, context maps, open-question registers, assumptions, work disaggregation, peer PRD/PLAN review, and execution briefs.

**Lab:** turn a demo ticket into an implementation plan, work package map, peer review notes, and Day 3 execution brief.

## Day 3 — Code Review, Testing, and Verification

Participants use the Day 2 execution brief to implement code in controlled phases, inspect the diff, run adversarial review, create a test strategy, add tests, fix failures, and produce PR-ready evidence.

**Lab:** implement a discount-code feature in the demo app, add tests, run verification, and create a reusable verification gate.

## Day 4 — Operating at Expert Speed

Participants review the run, analyze what worked, identify waste, improve reusable skills/gates, route work across Claude Code, background agents, scripts, or other coding agents, and build a personal/team operating checklist.

**Lab:** perform a blameless post-run review and improve one reusable workflow artifact.
