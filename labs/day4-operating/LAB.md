# Day 4 Lab — Post-Run Review and Workflow Improvement

## Objective

Turn the Day 2-3 run into reusable improvement.

## Steps

1. Review the plan, diff, tests, and verification output.
2. Classify contribution: delivery, learning, verification, leverage, waste.
3. Identify one improvement to the planning skill.
4. Identify one verification gate.
5. Decide whether part of the work should become a slash command, skill, custom agent, hook, or workflow.

Complete `post-run-review.md`.
