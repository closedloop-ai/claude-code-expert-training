# Lab 06 — Claude in Existing Workflows

## Objective

Learn how to use Claude Code effectively inside the workflow you already use.

## Tracks

Choose one primary track and compare it with one secondary track:

- terminal-first
- IDE-first
- `tmux`
- SSH / remote shell

## Exercises

1. Describe your normal workflow before Claude.
2. Identify where Claude helps most: planning, execution, review, verification, or investigation.
3. Identify one place where your normal tool remains better than Claude.
4. Design a workflow loop that combines both cleanly.
5. Write down the permissions, artifacts, and verification steps that fit this workflow.

## Deliverable

Produce a short workflow playbook:

- tools used
- where Claude enters the loop
- where human inspection remains primary
- what evidence gets captured
- what permissions are needed
