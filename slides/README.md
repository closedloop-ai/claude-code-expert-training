## How to Use These Files

Each day guide is written as an instructor deck outline, not as a finished slide export.

Each file includes:

- session purpose
- time blocks
- slide sequence
- speaker-ready slide content
- instructor notes
- demo scripts
- teaching prompts
- demo ideas
- demo rubric
- supporting course files

The `demo-artifacts/` directory contains ready-made examples instructors can open directly during class.

## Delivery Notes

- Start each major concept with the cohort prompt: `Has anyone tried this? What is working?`
- Treat the labs as applied work blocks inside the teaching day, not as the entire course structure.
- Preserve artifact handoffs across days so each day builds on the work produced earlier in the course.
